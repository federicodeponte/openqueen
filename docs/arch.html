Concurrency Model

Parallel — different projects

Project A and Project B start immediately in parallel. Each gets its own RUNNING-{task}.lock and a dedicated monitor.py. No coordination needed.

Queued — same project

A second task for Project A enters QUEUE.json. When the first task finishes, monitor.py calls dequeue_and_start() automatically.
